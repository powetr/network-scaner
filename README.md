# network-scaner
Small utility [bash] for host network scan with history reporting and zabbix-compatible export

1. ОБЩЕЕ ОПИСАНИЕ
   
Скрипт предназначен для системных администраторов и инженеров безопасности.
Позволяет проводить инвентаризацию сети, отслеживать изменения MAC-адресов
и анализировать задержки (ping) в исторической перспективе.

2. СИНТАКСИС КОМАНД

./nscaner -i <интерфейс> [опции]

3. ПОДРОБНОЕ ОПИСАНИЕ КЛЮЧЕЙ

СЕТЕВЫЕ НАСТРОЙКИ:

  -i <iface>      Интерфейс (eth0, wlan0 и т.д.). Обязателен для скана.
  
  -S <ip>         Сканировать только один указанный IP-адрес.
  
  -t <threads>    Количество потоков (по умолчанию 50).
  
  -p <ms>         Порог пинга. Пинг выше этого значения подсветится красным.

ЛОГИКА ИМЕН ХОСТОВ (-n):

  -n              Вывод только устройств, у которых определено имя хоста.
  
  -n all          Вывод всех активных устройств. Имя станет первым столбцом.
  
  -n <string>     Фильтр: вывод только тех, чье имя содержит <string>.

ФОРМАТИРОВАНИЕ MAC-АДРЕСОВ (-m):

  -m 1            MAC-адрес в ВЕРХНЕМ регистре (например, AA:BB:CC:11:22:33).
  
  -m 2            Удаление разделителей (например, aabbcc112233).
  
  -m 3            ВЕРХНИЙ регистр и без разделителей (например, AABBCC112233).
  
  * Примечание: Для режима анализа (-A) по умолчанию всегда включен "-m 1".

АНАЛИЗ ИСТОРИИ (-A):

  -A <ip>         Выводит историю активности узла из накопленных логов.Считает средний пинг, пиковый пинг и их соотношение.
  
  -d <days>       Фильтр по дням. При "-d 1" покажет данные за сегодня и вчера.При работе с ключом -X задает срок хранения логов.

ВЫВОД И ЛОГИ:

  -c              Включить цветную подсветку (Online - зеленый, High Ping - красный).
  
  -f <format>     Формат вывода: text (обычный) или json (для Zabbix LLD).
  
  -s [host]       Логирование в Syslog (локальный или на удаленный IP-адрес).
  
  -q              Alert Mode: выводит в консоль только факты смены MAC-адресов.
  
  -L              Limit Mode: показывает только те хосты, чей пинг выше порога (-p).

ОБСЛУЖИВАНИЕ БАЗЫ:

  -E              Экспорт: собирает все файлы из папки scans/ в один большой файл.
  
  -X              Принудительная очистка базы от файлов старше N дней (ключ -d).

4. ЛОГИКА АЛЕРТОВ (БЕЗОПАСНОСТЬ)

Если скрипт обнаруживает, что у IP-адреса изменился MAC-адрес по сравнению с предыдущим сканированием, создается файл "alerts_YYYY-MM-DD_HH_MM.log".
Логика сравнения игнорирует регистр и наличие двоеточий, поэтому смена формата вывода не вызывает ложных уведомлений.

5.СИСТЕМНЫЕ ТРЕБОВАНИЯ
 
ОС: Linux (Debian, Ubuntu, CentOS и др.)
Зависимости: iproute2, coreutils, iputils-ping, bc, awk.
